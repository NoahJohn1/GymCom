Expo Development Builds — Setup Guide

GymCom · Generated March 21, 2026
1 What Are Dev Builds?

Expo Go is a pre-built universal app with a fixed set of native modules. It's fast for prototyping but limited — some native packages like expo-notifications were removed since SDK 53.

Development Build is a custom-compiled app that includes your project's specific native dependencies. You compile it once, then develop against it identically to Expo Go (QR code scanning, hot reload, etc.).

Key difference: Expo Go = "one size fits all" → Dev Build = "compiled just for your project"

When to Rebuild

  • Rebuild required: Adding or removing native dependencies (e.g. expo-notifications, expo-camera)
  • No rebuild needed: JavaScript/TypeScript changes, styling, component updates, new screens

Rule of thumb: if npx expo install installs something with native code, rebuild. Otherwise, hot reload handles it.

3 Project Configuration
eas.json — Build Profiles

The development profile enables the dev client and produces a directly installable APK for Android.

{
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal",
      "android": { "buildType": "apk" },
      "env": { "NPM_CONFIG_LEGACY_PEER_DEPS": "true" }
    }
  }
}
Why NPM_CONFIG_LEGACY_PEER_DEPS? GymCom uses react@19.1.0. Many packages have peer conflicts with react-dom. This flag bypasses those without breaking functionality.
app.json — Native Plugins

Any native module needs its plugin registered under "plugins" in app.json.

"plugins": [
  "expo-router",
  ["expo-audio", { "microphonePermission": false }],
  ["expo-notifications", {
    "sounds": ["./assets/sounds/timer_done.mp3"]
  }]
]
Android naming rule: Resource names must use underscores only — no hyphens. Use timer_done.mp3 not timer-done.mp3.
4 Building for Android
1 eas build — Build the APK 
eas build --platform android --profile development

EAS uploads your project and builds in the cloud (5–15 minutes). On completion, you get a download link for the APK.

2 Install on device
  • Open the download link on your Android phone
  • Enable "Install from unknown sources" in Settings if prompted
  • Install the APK
3 Connect and develop 
npx expo start

Open the dev build app on your phone and scan the QR code — same workflow as Expo Go.

5 Building for iOS
Simulator 
eas build --platform ios --profile development

Produces a .app for the iOS simulator. Drag and drop onto the simulator to install.

Physical Device
  • Requires Apple Developer account ($99/year)
  • EAS handles provisioning profiles automatically
  • Register devices with eas device:create
  • Build with: eas build --platform ios --profile development
  • Install via QR code link from EAS
6 Both Platforms at Once 
eas build --platform ios --profile development
eas build --platform android --profile development
  • Each produces a separate installable artifact
  • Both connect to the same npx expo start dev server
  • Hot reload works on both simultaneously
7 Production Builds & Store Submission 
# Production builds
eas build --platform ios --profile production
eas build --platform android --profile production

# Submit to stores
eas submit --platform ios
eas submit --platform android
  • EAS handles code signing (iOS) and keystores (Android)
  • app.json must have ios.bundleIdentifier and android.package configured
  • GymCom already has both: com.gymcom.app
8 Common Issues
Android Resource Name Errors

Android resource names only allow lowercase letters, numbers, and underscores.

  • Rename files like timer-done.mp3 to timer_done.mp3
  • Update all code references to match
Notification Channels (Android 8+)
  • Notification sounds are tied to channels, not individual notifications
  • Create channels with Notifications.setNotificationChannelAsync() specifying the sound
  • Channels are cached by the OS — uninstall and reinstall if changing channel config
Expo Go Incompatibility

Some native modules don't work in Expo Go (SDK 53+). Use a try/catch to gracefully degrade:

let Notifications = null;
try { Notifications = require('expo-notifications'); } catch {}

App works in Expo Go without notifications; full functionality in dev build.
